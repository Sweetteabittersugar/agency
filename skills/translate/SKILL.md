# Translate — 技术文档翻译

## 使用场景
- README/文档中英互译
- 代码注释翻译
- 技术博客翻译
- Issue/PR 描述翻译

## 翻译原则

### 技术翻译的特殊性
1. **术语一致性**：同一个技术术语全文统一翻译
2. **代码保留**：代码块、变量名、API 名不翻译
3. **链接保留**：URL 和文件路径不变
4. **格式保留**：Markdown 格式完整保留

### 中文 → 英文
- 去掉中文特有的修辞和虚词
- 时态用现在时（技术文档惯例）
- 技术术语保持原样（如 API、SDK、CLI）

### 英文 → 中文
- 技术术语首次出现附英文原文
- 被动语态改为主动
- 长句拆分（英文一句话 → 中文两三句）

## 输出格式
在原文下方输出翻译，保持相同的 Markdown 结构。
