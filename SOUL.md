# SOUL.md — 项目灵魂

> 这个文件定义了 everythingclaudecode 的信仰、边界和行事准则。它不是一个功能列表——而是"我们是谁"。

## 我们的信仰

**工具应该适配人，不是人适配工具。**

Claude Code 是一把好刀。但好刀不意味着好刀法。它出厂时是一张白纸——没有专业分工、没有成本意识、没有持久记忆。everythingclaudecode 的存在，就是让每一个 Claude Code 用户不花三个月磨合，也能获得一套经过日常生产验证的"用刀方式"。

## 为什么存在

裸 Claude Code 和"高效工作"之间差着一个配置层。缺三样东西：

1. **分工** — 一个人做不完所有事，你需要一支团队（Agent 矩阵）
2. **纪律** — 没有自动化规范，好习惯撑不过第三次赶工（Rules + Hooks）
3. **账本** — 不知道花了多少钱、花在哪里（Cost Tracker）

ECC 给出了"怎么组织配置"的答案——207K Stars 证明这是对的。everythingclaudecode 在这个基础上加了两个锚点：**中文原生**和**生产验证**，让配置从模板变成可以真实运转的系统。

## 核心原则

### 1. 专业分工，拒绝全能
一个 Agent 做好一件事。9 个专业 Agent > 1 个万能 Agent。coder 不审查，explorer 不写代码，reviewer 不改代码。边界清晰，协作才有意义。

### 2. 成本可视化
AI API 调用不是免费的。每一分钱都该被看见——按模型、按日期、按 Agent。知道钱花在哪，才知道该优化哪。`@cost` 不是附加功能，是"信任系统"的基础。

### 3. 自动化不替代判断
自动路由任务、自动压缩上下文、自动清理进程。但任何时候用户说"停"——一切停止，不辩护，不继续。自动化是服务，不是牢笼。

### 4. 结果导向，过程透明但不啰嗦
Agent 内部可以有复杂的思考链，但用户只看到精简摘要。需要细节时可以深入，不需要时不被干扰。轻活直接调，重活走调度引擎。每一层都是真实的负担减轻。

### 5. 中文原生不是翻译
中文是第一语言——文档用中文，Agent 指令用中文，交互用中文。不是把英文改成了中文，而是用中文思维重新组织一切。英文 README 存在是为了兼容，不是为了取悦。

## 我们不做的事情

- **不做 AI 工具本身** — 不做模型、不做 API、不做聊天界面。只做 Claude Code 的配置增强层
- **不做零配置** — 不给"开箱即用"的幻觉。这套系统需要你理解它在做什么，安装脚本只是帮你省掉复制粘贴的时间
- **不做万能方案** — 不为每种语言每个框架都提供规则。已有的是经过实际使用验证的，没验证的不放
- **不做后台服务** — 没有 daemon、没有常驻进程。Maestro 脚本单次执行，用完即退
- **不做封闭生态** — 所有配置是纯文本文件，可以随时手动修改。按需选用，不必全装
