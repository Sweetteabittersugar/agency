## 〇、教育助力层 — "我不懂，但我能用"

> 核心洞察：AI 编程工具行业月月在变。Agent、MCP、Hook、Skill——这些概念三个月前还不存在。业内精英也在边用边学。不能假设用户懂。

### 0.1 术语场景化解释

每个专业术语首次出现时带 `?` 图标，点击出场景化解释，不说教科书定义：

| 术语 | 不用 | 用 |
|------|------|-----|
| Agent | "具备特定能力的 AI 角色" | "想象有 31 个助手，各有所长。你说要干什么，Agency 自动派最合适的" |
| Skill | "可复用的工作流模板" | "比如每次写完代码都要查安全漏洞。Skill 把这个流程写成模板，Agent 在合适时候自动调用，不用每次重复吩咐" |
| MCP | "Model Context Protocol" | "Agent 本来只能聊天。装上 MCP 后能操作浏览器、搜网络、查文档——像给助手配了电脑和网线" |
| Profile | "上下文加载策略" | "轻量=快+便宜+简单任务；全功能=慢+强+复杂项目。随时切换" |
| Hook | "生命周期自动化脚本" | "像手机闹钟——设'开机自动检查环境''关机前保存日志'" |
| 路由 | "Agent 自动匹配" | "你说'帮我修 Bug'，系统自动判断派给 coder 还是 debugger，不用你记 31 个名字" |

### 0.2 功能引导卡片

每个核心功能首次打开不是空白页，先弹引导卡片：标题 + 能干什么 + 适合什么 + 不适合什么 + [试试看] [了解更多] [暂时不用] 三个按钮。覆盖：智能调度、Agent 工厂、Skill 编辑、MCP 配置、仪表盘。

### 0.3 交互式概念图谱

帮助面板新增"概念图谱"标签页，树形图展示所有概念关系。点击任何节点跳转到对应功能。

### 0.4 路由决策透明化

```
当前：   📍 代码开发 → 🤖 coder (85%)
改为：   📍 代码开发 → 🤖 coder (85%)
        💡 你的任务是"写代码"，所以派给了 coder。
        如果要审查代码，说"审查这段代码"→ 自动派给 reviewer。
        [换个 Agent]
```

### 0.5 按场景导航

侧边栏新增"场景"Tab：我遇到了 Bug / 我想写新功能 / 我想理解项目 / 我想检查代码质量 / 我想写测试 / 我想看费用报告 / 我想搜索资料——用户只需描述意图，不用知道 Agent 名字。

### 0.6 行业动态提示

帮助面板新增"最近更新"：模型降价/新功能上线/推荐配置/使用技巧，每月 2-3 条。

## 一、上手体验

### 1.1 Demo 模式从一句话变成交互式沙盒

当前：点跳过 → 一段文字。改为：进入后侧边栏有假数据（3 Agent + 5 Skill + 2 历史），聊天区有欢迎消息 + 3 个快捷任务按钮（"帮我写网页爬虫""审查这段代码""搜索 TODO"），仪表盘有模拟图表。点任何任务 → "🔧 配置 API Key 即可执行真实任务" + 一键跳转设置。参考 Figma Playground / Cursor "Try with example project"。

### 1.2 Provider 从 11 项下拉变成 2 个问题

Q1: 你在哪个地区？→ 中国大陆推 DeepSeek/通义/Kimi；海外推 Anthropic/OpenAI/Google。Q2: 你的预算？→ 免费推 DeepSeek；中等推 V4-Pro；性能优先推 Claude Opus。附注册链接和预计月费。保留 [查看全部 11 个 Provider] 折叠选项。参考 Notion 的偏好式引导。

### 1.3 渐进式功能暴露

Day 0: 聊天单面板 + Key 配置。Day 1-2: 仪表盘 + Agent 列表。Day 3-5: 智能调度 + 多面板。Day 7+: Agent 工厂 + Skill 编辑 + Profile。设置页有"解锁全部"开关。参考 Superhuman（分阶段 onboarding 完成率提升 40%）。

### 1.4 智能错误恢复

不只是报错+解释，自动修复：Key 无效→弹出输入框；网络断→自动重试 3 次(间隔 2/5/10s)；Claude CLI 未安装→一键安装按钮；MCP 离线→自动重启；Agent 超时→自动重试 1 次(降 temperature)。参考 Chrome 错误页。

## 二、默认设定 — 开箱即用

- **智能 Provider**：检查 .env → 检测 IP 地区 → 推荐对应 Provider
- **自动检测**：Claude CLI 路径(扫 PATH + npm + 常见路径)、Python 版本、当前目录(.git → 自动设项目目录)、MCP 运行状态、已有 Agent(合并显示不覆盖)
- **Profile 自动选**：Day 0-2→minimal(只有输入框)；Day 3+→standard(Agent 列表+路由)；手动升 full
- **预设模板**：输入框旁"模板"下拉，6 个常用场景一键填入，用户可保存自定义模板

## 三、操作简化

### 3.1 模糊意图解析

"爬一下这个网站"→自动补全为"@coder 写网页爬虫"；"太慢了"→引用上一条输出+"@performance-optimizer 分析瓶颈"；"翻成英文"→引用上一条+"translate"；"改一下"→弹出上次输出+让用户指定改什么。

### 3.2 批量操作

多选删除 Agent、清空全部历史(带撤销)、全局 Agent 粘性、聊天输入 `/path` → 文件选择器。

### 3.3 键盘快捷键全覆盖

Ctrl+K 命令面板 / Ctrl+N 新面板 / Ctrl+W 关面板 / Ctrl+, 设置 / Ctrl+/ 快捷键帮助 / Ctrl+Enter 发送 / Esc 停止 / 1-5 切仪表盘标签。

### 3.4 命令面板

按 Ctrl+K 弹出搜索框，输入 "agent"→31 Agent、"skill"→40+ Skill、"theme"→换主题、"export"→导出配置、"clear"→清缓存。参考 VS Code Command Palette。

## 四、防误触

### 4.1 三级保护

| 级别 | 示例 | 方式 |
|------|------|------|
| 安全 | 发消息、切面板、看 Agent | 零摩擦 |
| 可恢复 | 删历史、禁用 Skill、关面板 | 执行 + Toast + 5秒撤销按钮 |
| 不可逆 | 删 Agent/Skill、清全部历史 | 弹窗 + 按钮默认选取消 |

核心：80% 操作不需要弹窗。参考 Gmail Undo Send / GitLab destructive action 分级。

### 4.2 撤销系统

删对话 → "已删除 [撤销]"；关面板 → "已关闭 [恢复]"；清提示词 → "已清空 [恢复]"；禁用 Skill → "已禁用 [启用]"；改配置 → "已保存 [回滚]"。实现：前端乐观更新 + 5 秒延迟后端写入。

### 4.3 输入保护

空消息禁发(按钮置灰)、失焦草稿保存 localStorage、粘贴大文本提示确认、有未完成任务时关闭页弹窗确认、Key 输入框 password+显示切换、Provider 切换提示重输 Key、删除最后 3 个 Agent 阻止。

### 4.4 视觉反确认

删除按钮红色+悬停加深、按钮文字用动词("删除这个 Agent"而非"确定")、0.3s 动画延迟防双击、批量删除显示"已选 3 个，将永久删除"。

## 五、自定义能力

### 5.1 三级体系

L1 即用型(所有人)：切换主题/字体/布局/Profile。L2 配置型(进阶)：Agent 工厂+Skill 编辑+MCP 开关。L3 代码型(开发者)：直接编辑 .md/.py/路由权重。

### 5.2 自定义清单

外观：三主题、字体三档、侧边栏位置、消息密度、时间戳格式、代码块主题。行为：发送键、自动滚动、通知方式、面板动画、消息排队、自动保存间隔。工作流：默认 Agent、默认 Provider、输出目录、自定义模板、自定义快捷键。

### 5.3 配置导入导出

导出全部配置→下载 agency-config.json；导入→冲突时差异对比逐项选择；重置为默认→保留 API Key。

## 六、视觉与无障碍

- **对比度审计**：var(--muted)≥4.5:1、边框 hover 变化、错误/成功色加图标冗余编码、focus 态品牌色 2px outline
- **动效**：功能性 150-200ms / 反馈性 100ms / 无装饰性动效 / 提供减少动画开关 / prefers-reduced-motion 自动检测
- **响应式三档**：桌面 (>1024px) 侧边栏固定+多面板、平板 (768-1024) 侧边栏可折叠+最多2面板、手机 (<768) 侧边栏隐藏+底部Tab+单面板
- **高对比度主题**：纯黑背景+纯白文字、蓝色链接+黄色焦点框、无半透明/渐变/阴影、边框≥2px

## 七、反馈与引导

| 位置 | 当前 | 改为 |
|------|------|------|
| 聊天区空 | "⚡ 就绪" | "👋 试试：帮我写个网页爬虫" + 3 快捷按钮 |
| Agent 列表失败 | "加载失败" | "无法连接服务器 [重试] [检查服务状态]" |
| 仪表盘空 | 空图表 | "还没有数据。发一个任务后这里会有实时统计 📊" |
| 历史空 | "暂无历史" | "对话会出现在这里。试试发第一条消息 👆" |
| Skill 空 | "暂无 Skills" | "Skills 是给 Agent 的工作流模板 [了解 →] [创建 →]" |

每个功能旁 `?` tooltip。操作进度可视化（面板标题闪烁+路由栏动画+五阶段进度条+排队计数）。刷新关闭保护（5s 草稿保存+未完成任务提醒+SSE 自动重连）。

## 八、从实际交互映射来的教训

| 我们踩过的坑 | UX 方案的应对 |
|-------------|-------------|
| Skill 编辑失败但前端静默无报错 | 操作失败必须可见反馈 + 自动重试（§1.4） |
| 改 sidebar.js 没检查影响，改坏其他功能 | 批量操作前展示影响范围预览（§4.4） |
| coder 关键词权重抢了其他 Agent 的活 | 路由决策带人话解释 + [换个 Agent] 按钮（§0.4） |
| `git checkout -- .` 丢 70 个文件 | 所有不可逆操作延迟执行 + Toast 撤销（§4.2） |
| 多次被要求"先出方案别动手" | 渐进式功能暴露——别一次性堆复杂功能（§1.3） |
| Linter 自动改代码无通知 | 任何自动修改都报告"改了什么、为什么" |
| start.bat 硬编码 D:\agency | 所有路径自动扫描，不假设用户环境（§2.2） |
| token_counter 模块缺失导致 import 报错 | 环境自检：SessionStart 扫描依赖完整性（§2.2） |
