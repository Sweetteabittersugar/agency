# Agency 用户体验全栈优化方案

> 2026-06-09 | 目标：从"开发者能用"到"任何人拿起来就爽"

---

## 用户画像

| 画像 | 特征 | 需求 |
|------|------|------|
| **AI 新手** | 刚接触 AI 编程，不懂 Agent/Skill/MCP，可能没用过 Claude Code | 一句话描述需求就能干活，不关心底层 |
| **效率型开发者** | 有 AI 工具经验，追求速度，讨厌多余步骤 | 快捷键、批量操作、管道式工作流 |
| **架构师/Leader** | 管理多项目多团队，关注成本和质量 | 全局视图、审计日志、权限控制、费用看板 |
| **移动端用户** | 手机/平板操作，屏幕小、触控精确度低 | 大按钮、防误触、消息排队、离线通知 |

## 〇、教育助力层 — "我不懂，但我能用"

> AI 编程工具行业月月在变。Agent、MCP、Hook、Skill、Embedding、Profile——这些概念三个月前还不存在，现在已经是基础设施。业内精英也在边用边学。**Agency 的任务不是假设用户懂，而是帮用户在用的过程中自然学会。**

### 0.1 每个概念都有一个"这是啥"

所有专业术语首次出现时，旁边带一个 `?` 图标。点击不是弹出冷冰冰的定义，而是**场景化解释**：

| 术语 | 传统解释 | Agency 的场景化解释 |
|------|---------|-------------------|
| Agent | "具备特定能力的 AI 角色" | "想象你有 31 个助手，有的擅长写代码、有的擅长找 Bug、有的擅长写文档。你只需描述任务，Agency 自动派给最合适的助手。" |
| Skill | "可复用的工作流模板" | "比如你每次写完代码都要检查一遍安全漏洞。Skill 就是把这个流程写成模板——Agent 在合适的时候自动调用，不用你每次重复吩咐。" |
| MCP | "Model Context Protocol" | "Agent 本来只能聊天。装上 MCP 后它能操作浏览器、搜索网络、查最新文档——就像给助手配了电脑和网线。" |
| Profile | "上下文加载策略" | "轻量模式只加载必要的，适合简单任务（快+便宜）。全功能模式加载全部，适合复杂项目（慢+强）。你可以随时切换。" |
| Hook | "生命周期自动化脚本" | "就像手机闹钟——可以设"开机时自动检查环境""关机前自动保存日志"。不需要手动操作。" |
| 路由 | "Agent 自动匹配" | "你说"帮我修 Bug"，Agency 自动判断应该派给 coder 还是 debugger。你不需要知道 31 个 Agent 各自擅长什么——系统会选。" |

### 0.2 功能入口带"新手引导卡片"

关键功能第一次打开时，不是直接显示空白界面，而是**先弹一张引导卡片**：

```
┌─────────────────────────────────┐
│  🧠 智能调度                     │
│                                 │
│  这是 Agency 最核心的能力。       │
│  你把一个大任务丢进来——           │
│  比如"帮我做一个用户登录系统"——   │
│  Agency 会自动：                 │
│                                 │
│  1. 拆成小任务（设计数据库、      │
│     写接口、写前端页面…）         │
│  2. 分配给最合适的 Agent         │
│  3. 逐个执行，汇总结果           │
│                                 │
│  适合：复杂项目、多步骤任务       │
│  不适合：简单问答、单文件修改     │
│                                 │
│  [试试看]  [了解更多]  [暂时不用] │
└─────────────────────────────────┘
```

每个核心功能都有一张这样的卡片。用户可以关掉，但至少看过一次。

### 0.3 概念图谱 — "我在哪，我能干什么"

在帮助面板新增一个 "概念图谱" 标签页，用简单的**树形图**展示所有概念的关系：

```
Agency
├── 🤖 Agent（执行任务的 AI 角色）
│   ├── 系统自动选（路由）← 大部分情况用这个
│   ├── 手动指定 @agent名 ← 你知道该用谁时
│   └── 创建自己的 Agent（Agent 工厂）← 进阶
├── 📋 Skill（给 Agent 的工作模板）
│   ├── 内置 40+ 个 ← 覆盖常见场景
│   ├── 启用/禁用 ← 按需开关
│   └── 编辑源码 ← 自定义行为
├── 🔌 MCP（给 Agent 的外部工具）
│   ├── Playwright → 操作浏览器
│   ├── Context7 → 查最新文档
│   └── Brave Search → 搜索网络
├── 🧠 智能调度（复杂任务自动拆解）
├── 📊 仪表盘（费用/权限/状态监控）
├── ⚙️ Profile（轻量/标准/全功能切换）
└── 🔔 Hook（启动/关闭/压缩时的自动操作）
```

这个图是**交互式的**——点击任何节点跳转到对应功能页面或设置面板。

### 0.4 路由决策透明化

当前路由栏显示："📍 代码开发 → 🤖 coder"。对新手来说这是黑盒。

改为**带解释的路由信息**：

```
当前：  📍 代码开发 → 🤖 coder (85%)
改为：  📍 代码开发 → 🤖 coder (85%)
       💡 你的任务是"写代码"，所以派给了 coder。
       如果要审查代码质量，可以说"审查这段代码"→ 自动派给 reviewer。
       你也可以手动换 Agent → [换个 Agent]
```

每次路由决策都附带一句**人话解释**——为什么选这个 Agent、什么情况下会选别的。

### 0.5 按场景引导，不按功能分类

当前侧边栏按功能分类：Agent / Skills / 历史。新用户不知道"Agent"和"Skill"的区别。

新增一个 **"场景"标签页**，按用户意图组织：

```
侧边栏新增 "场景" Tab：
├── 🐛 我遇到了一个 Bug → 自动 @coder 或 @debugger
├── 📝 我想写新功能 → 自动打开智能调度模式
├── 🔍 我想理解一个项目 → 自动 @explorer
├── ✅ 我想检查代码质量 → 自动 @reviewer
├── 🧪 我想写测试 → 自动 @test-runner
├── 📊 我想看费用报告 → 打开仪表盘
├── 🔒 我想检查安全漏洞 → 自动 @security-reviewer
└── 🌐 我想搜索资料 → 自动启动 MCP 搜索
```

用户不需要知道 Agent 名字——只需要描述"我想干什么"。系统把意图映射到功能。

### 0.6 行业动态提示

帮助面板新增 "最近更新" 标签页：

```
├── 📰 模型动态：DeepSeek V4 降价 75%（2026-05-31）
├── 🆕 新功能：五阶段管线已上线（智能调度升级）
├── 🔧 推荐配置：本月最受欢迎的 Provider 是 DeepSeek
└── 💡 技巧：试试用 @explorer 分析陌生项目，比手动翻文件快 10 倍
```

每月更新 2-3 条。让所有用户——无论新手还是老手——都能跟上行业变化。

## 一、上手体验 — 从"5 分钟跑通"到"30 秒爱上"

### 1.1 Demo 模式升级

当前：点"跳过"后显示一段文字。改为 **交互式 Demo 沙盒**。

```
Demo 模式进入 → 自动创建一个模拟项目
├── 侧边栏已有假数据（3 个 Agent + 5 个 Skill + 2 条历史）
├── 聊天区已有一条欢迎消息 + 3 个快捷任务按钮
│   ├── [🧪 帮我写个网页爬虫]
│   ├── [📝 审查这段代码]
│   └── [🔍 搜索项目中所有 TODO]
├── 仪表盘有模拟数据（假费用图表、假权限日志）
└── 点击任何快捷任务 → 弹出 "🔧 配置 API Key 即可执行真实任务" + 一键跳转设置

价值：用户在没有 Key 的情况下 30 秒就能理解产品是干什么的
```

参考：Cursor 的 "Try with example project"、Figma 的 "Playground file"

### 1.2 智能推荐 Provider

当前：向导直接显示 11 个 Provider 下拉框。改为 **问答式推荐**。

```
向导 Step 1 改为两个问题：
Q1: 你在哪个地区？
  ○ 中国大陆 → 推荐 DeepSeek / 通义千问 / Kimi（便宜+中文好+无需翻墙）
  ○ 海外 → 推荐 Anthropic / OpenAI / Google
  ○ 不限

Q2: 你的预算是？
  ○ 免费/低成本 → DeepSeek（$0.14/M token）/ 硅基流动（有免费额度）
  ○ 中等 → DeepSeek V4-Pro（$0.435/M）
  ○ 性能优先 → Anthropic Claude Opus

→ 根据答案推荐 1-2 个 Provider，附注册链接和预计月费
→ 仍然保留 [查看全部 11 个 Provider] 折叠选项
```

参考：Notion 的 "What will you use Notion for?" / Spotify 的偏好选择

### 1.3 渐进式功能暴露

**原则：新用户只看到 20% 的核心功能。随着使用深度逐步解锁。**

| 使用天数 | 自动解锁 |
|---------|---------|
| Day 0 | 聊天（单面板）+ API Key 配置 |
| Day 1-2 | 仪表盘概览 + 侧边栏 Agent 列表 |
| Day 3-5 | 智能调度 + 多面板分屏 |
| Day 7+ | Agent 工厂 + Skill 编辑 + Profile 切换 |
| 手动开启 | MCP 配置 / 远端访问 / Hook 配置（标记 🧪 实验性） |

实现方式：localStorage `agency_unlocked_features`，首次使用后按时间逐步解锁。设置页有"提前解锁全部功能"开关。

参考：VS Code 的 "Getting Started" 渐进引导 / Superhuman 的分阶段 onboarding

### 1.4 智能错误恢复

每一条错误信息不只是"出错了+原因+方案"，还要**自动尝试修复**：

| 错误类型 | 自动恢复策略 |
|---------|-------------|
| API Key 无效 | 弹出 Key 输入框，自动选中当前 Provider |
| 网络连接失败 | 自动重试 3 次（间隔 2s→5s→10s），失败后提示切换网络 |
| Claude CLI 未安装 | 检测是否 npm 可用 → 提示一键安装命令 → [帮我安装] 按钮 |
| MCP 服务离线 | 自动重启 MCP 进程，失败后提示关闭该服务 |
| Agent 超时 | 自动重试 1 次（降低 temperature），再失败才问用户 |

参考：Chrome 的 "Something went wrong" 错误页 — 不仅解释，还提供具体行动按钮

## 二、默认设定 — 开箱即用，零配置启动

### 2.1 智能默认 Provider

```
默认 Provider 检测逻辑：
1. 检查 .env 中是否有已配置的 Key → 直接启用
2. 检查用户 IP 地区 → 中国大陆 → DeepSeek；海外 → Anthropic
3. 都不满足 → Demo 模式，引导页推荐 DeepSeek
```

### 2.2 自动检测与预填充

| 检测项 | 自动操作 |
|--------|---------|
| `claude` CLI 路径 | 自动扫描 PATH + npm global + 常见安装路径 |
| Python 版本 | 提示升级或降级建议 |
| 当前目录 | 自动设为项目目录（如检测到 .git） |
| MCP 服务器 | 扫描用户级 + 项目级 `.mcp.json`，自动检测运行状态 |
| 已有 Agent | 扫描 `~/.claude/agents/`，合并显示而非覆盖 |

### 2.3 Profile 自动选择

```
首次使用后的 Profile 自动策略：
├── Day 0-2: minimal（只加载 router agent，0 skill）
│   └── 用户只看到一个输入框 + 一个回复区
├── Day 3+: standard（领域全组 + 按需 skill）
│   └── 侧边栏出现 Agent 列表，聊天区出现路由信息
└── 用户手动升级 full（全部功能）
```

### 2.4 预设工作流模板

聊天输入框旁新增 "模板" 下拉：

```
常用模板（一键填入输入框）：
├── 🔍 "帮我分析 @explorer 这个项目的整体结构"
├── 🐛 "帮我修复 @coder 这个 bug: [描述]"
├── 📝 "帮我写 @coder 一个单元测试: [函数名]"
├── 🧹 "@reviewer 审查最近的改动"
├── 📊 "@cost-analyst 分析本周 API 费用"
└── 🌐 "帮我 @explorer 搜索项目中所有 TODO 和 FIXME"

用户可以保存自己的常用模板
```

参考：Raycast 的 Quicklink / Notion 的 Template Button

## 三、操作简化 — 能点的不让打，能打一次的不让打两次

### 3.1 一句话操作

```
当前：用户必须写 "帮我写一个排序函数"
增强：自然语言模糊意图也能处理

输入示例 → 自动解析：
"爬一下这个网站" → 自动补全为 "@coder 写一个网页爬虫"
"太慢了" → 自动引用上一个 Agent 输出 + "@performance-optimizer 分析性能瓶颈"
"翻成英文" → 自动引用上一次输出 + "translate to English"
"改一下" → 弹出上次输出 + 让用户指定改什么
```

### 3.2 操作批量化

| 当前 | 改为 |
|------|------|
| 删 Agent 逐个确认 | 多选 → 一键删除 |
| 清历史一条条删 | "清空全部历史"按钮（带撤销）|
| 每个面板独立配 Agent | 全局 Agent 粘性：[@coder] 标签固定在输入框上 |
| 手动输入文件路径 | 聊天输入框中 `/path` → 弹出文件选择器 |

### 3.3 键盘快捷键全覆盖

```
全局：
  Ctrl+K / Cmd+K → 命令面板（快速搜索任何功能）
  Ctrl+N / Cmd+N → 新面板
  Ctrl+W / Cmd+W → 关闭当前面板
  Ctrl+Shift+P → Profile 切换
  Ctrl+, → 打开设置
  Ctrl+/ → 快捷键帮助

面板内：
  Ctrl+Enter → 发送
  Shift+Enter → 换行
  Ctrl+↑↓ → 切换历史消息（重新编辑上一条）
  Esc → 停止生成 / 取消输入

仪表盘：
  1-5 → 切换标签页（概览/权限/SubAgent/Hooks/MCP/环境/测试）
```

### 3.4 命令面板（Ctrl+K）

参考 VS Code Command Palette 和 Raycast：

## 四、防误触 — 不打断心流，但保护关键操作

### 4.1 三级操作保护

| 级别 | 操作示例 | 保护方式 |
|------|---------|---------|
| **安全** | 发送消息、切换面板、查看 Agent | 无保护，直接执行 |
| **可恢复** | 删除历史、禁用 Skill、关闭面板 | 执行 + Toast 通知 + 5 秒 "撤销"按钮 |
| **不可逆** | 删除 Agent、删除 Skill、清空全部历史 | 确认弹窗（按钮默认选中"取消"） |

**核心原则：80% 的操作不需要确认弹窗。弹窗是最后手段。**

参考：Gmail 的 Undo Send（延迟执行+撤销）/ GitLab 的 destructive action 分级 / macOS Finder 的"移到废纸篓"可撤销

### 4.2 撤销系统

```
可撤销的操作（5-10 秒窗口）：
├── 删除对话 → "对话已删除 [撤销]"
├── 关闭面板 → "面板已关闭 [恢复]"
├── 清除 Agent 提示词 → "已清空 [恢复]"
├── 禁用 Skill → "已禁用 [启用]"
└── 修改配置 → "配置已保存 [回滚]"

实现：前端乐观更新 + 延迟后端写入（5秒后再真正执行删除/修改）
```

### 4.3 输入保护

```
防止误发送：
├── 空消息不能发送（按钮置灰）
├── 输入框失焦后内容不丢失（自动保存草稿到 localStorage）
├── 粘贴大段文本时提示 "粘贴内容较长（xxx字），是否发送？"
└── 刷新/关闭页面时如有未完成的任务弹出 "有 1 个任务正在执行，确定离开？"

防止误配置：
├── API Key 输入框 type=password + 显示/隐藏切换
├── Provider 切换时提示 "切换 Provider 后需重新输入 Key"
└── 删除所有 Agent 时阻止（至少保留 3 个）
```

### 4.4 视觉反确认

危险按钮不靠弹窗保护，靠**视觉信号**让用户自然三思：

```
删除类操作：
├── 按钮红色背景 + 悬停加深
├── 按钮文字用动词描述后果："删除这个 Agent" 而非 "确定"
├── 点击后 0.3s 动画延迟（防止手抖双击）
└── 批量删除时显示 "已选 3 个，将永久删除 [取消] [确认删除 3 个 Agent]"
```

参考：macOS 的 "Empty Trash" 音效 + 动画 / GitHub 的 "Delete branch" 红色大按钮 / Arc 浏览器的 Archive Tab 动画延迟

## 五、自定义能力 — 给高手刀，给新手盾

### 5.1 三级自定义体系

| 级别 | 用户 | 能力 |
|------|------|------|
| **L1 即用型** | 所有人 | 切换主题/字体/布局/Profile，无需理解底层 |
| **L2 配置型** | 进阶用户 | Agent 工厂 + Skill 编辑 + MCP 开关，填写即生效 |
| **L3 代码型** | 开发者 | 直接编辑 Agent .md / Skill SKILL.md / Hook .py / 路由权重 |

### 5.2 可自定义清单

```
外观：
├── 亮色 / 暗色 / 高对比度 三主题
├── 字体大小（Ctrl+/- 已有，需加预设 小/中/大）
├── 侧边栏宽度（已有拖拽）
├── 侧边栏位置（左/右）
├── 消息密度（紧凑/标准/舒适）
├── 时间戳显示（相对/绝对/不显示）
└── 代码块主题（github-dark / monokai / one-dark-pro / dracula）

行为：
├── 发送键 Ctrl+Enter / Enter（已有 Shift+Enter 换行）
├── 自动滚动开关（新消息时是否自动滚到底部）
├── 通知方式（Toast / 无 / 声音）
├── 面板切换动画（开/关）
├── 消息排队（开/关，移动端默认开）
└── 自动保存间隔（10s/30s/60s/手动）

工作流：
├── 默认 Agent（所有任务自动 @xxx）
├── 默认 Provider + Model
├── 输出目录（已有）
├── 自定义模板（保存常用 prompt）
└── 自定义快捷键
```

### 5.3 配置导入/导出

```
设置页新增：
├── [导出全部配置] → 下载 agency-config.json
│   包含：Agent/Provider/模板/快捷键/外观设置
├── [导入配置] → 上传 agency-config.json
│   冲突时显示差异对比 → 用户逐项选择保留哪个

## 六、视觉与无障碍 — 看 3 秒不累，用 3 小时不烦

### 6.1 对比度合规

当前暗色主题需审计：

| 问题 | 修复 |
|------|------|
| 次要文本 `var(--muted)` 对比度可能不足 | 确保 ≥ 4.5:1（正文）/ ≥ 3:1（大文本） |
| 按钮边框 `var(--border)` 可能过淡 | 增加边框对比度，hover 时明显变化 |
| 成功/警告/错误色可能只靠颜色 | 增加图标前缀（✅/⚠️/❌）做冗余编码 |
| focus 态可能不可见 | 用品牌色 2px outline 替代浏览器默认虚线 |

### 6.2 动效适度

```
动效三原则：
├── 功能性动效（列表加载、消息出现）：150-200ms ease-out，快速但可感知
├── 反馈性动效（按钮点击、开关切换）：100ms ease，几乎瞬间
├── 装饰性动效：无。不做弹跳、旋转、彩虹渐变
└── 设置中提供 [减少动画] 开关（prefers-reduced-motion 自动检测）
```

### 6.3 响应式三档

```
桌面 (>1024px)：侧边栏固定 + 多面板分屏
平板 (768-1024px)：侧边栏可折叠 + 最多 2 面板
手机 (<768px)：侧边栏隐藏（底部 Tab 切换）+ 单面板全屏
```

### 6.4 高对比度主题

新增 "高对比度" 主题（除亮色/暗色外第三种选择）：
- 纯黑背景 `#000` + 纯白文字 `#fff`
- 蓝色链接 + 黄色焦点框
- 无半透明、无渐变、无阴影
- 所有边框 ≥ 2px

```

### 7.3 进度可感知

```
操作进度可视化：
├── Agent 执行中 → 面板标题闪烁 + 路由栏动画 + 五阶段进度条
├── 多任务排队 → 侧边栏显示 "排队中 (2/5)"

├── 配置保存 → [保存中…] → [✅ 已保存]（1.5s 后消失）
└── 长时间无响应 → "任务仍在执行中（已等待 3 分钟）… [取消等待]"
```

### 7.4 上下文不丢失

```
刷新/关闭保护：
├── 输入框草稿自动保存（每 5 秒写 localStorage）
├── 有未完成任务时关闭页面 → 弹出 "1 个任务正在执行，关闭后将丢失进度"
├── 重新打开时 → 恢复上次输入框内容 + 侧边栏状态 + 面板布局
└── 会话断连后自动重连 SSE，成功则显示 "已恢复连接"
```
