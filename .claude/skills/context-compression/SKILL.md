# 上下文压缩

## 用途
在上下文窗口接近上限时，自动压缩历史对话，保留关键信息不丢失。

## 核心规则
- 上下文超过窗口 75% 时触发压缩，生成结构化摘要替代原始对话
- 摘要保留：当前任务目标、已做出决策、待办事项、关键文件路径
- 代码片段引用文件路径而非原文，减少 token 消耗
- 压缩后必须保留用户最新完整指令（不截断最后一条消息）
- 压缩操作记录到日志，包含压缩前后 token 数对比
