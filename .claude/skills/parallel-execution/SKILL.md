# 并行执行

## 用途
识别可并行执行的独立任务，最大化吞吐量，减少总耗时。

## 核心规则
- 分析任务依赖关系，无依赖的独立任务全部并行执行
- 有 A→B→C 依赖链的任务保持串行，不可强行并行
- 并行任务数限制在 5-8 个，避免资源竞争
- 所有并行任务完成后统一汇总结果，按原始顺序报告
- 并行任务中任何一个失败，其余仍继续执行，最终汇总成功/失败列表
