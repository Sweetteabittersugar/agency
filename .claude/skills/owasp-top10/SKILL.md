# OWASP Top 10 安全检查

## 概述
基于 OWASP Top 10（2021版）的 Web 应用安全风险清单，提供每类风险的识别方法、攻击示例和修复方案。

## 使用场景
- 代码审查中的安全检查环节
- 上线前的安全审计
- 安全培训材料参考
- 渗透测试后的漏洞修复

## OWASP Top 10 (2021)
1. **A01: 访问控制失效** —— 越权、URL 篡改、CORS 配置错误
2. **A02: 加密机制失效** —— 明文传输、弱加密算法、硬编码密钥
3. **A03: 注入攻击** —— SQL 注入、命令注入、LDAP 注入
4. **A04: 不安全设计** —— 缺少输入验证、无速率限制、过度暴露错误信息
5. **A05: 安全配置错误** —— 默认密码、不必要的功能开启、错误配置云存储权限
6. **A06: 易受攻击的组件** —— 过时依赖、已知漏洞未修复、未扫描第三方包
7. **A07: 身份认证失效** —— 弱密码策略、会话固定、无 MFA
8. **A08: 软件和数据完整性失效** —— 不安全的 CI/CD、反序列化漏洞、未验证的更新
9. **A09: 日志和监控不足** —— 无审计日志、异常未告警、日志可被篡改
10. **A10: SSRF（服务端请求伪造）** —— 用户控制 URL、未验证重定向、内部网络暴露

## 核心原则
1. **纵深防御**：不依赖单一防护层。输入验证 → 参数化查询 → 最小权限 → 审计日志层层设防。
2. **最小权限**：服务和用户只拥有完成任务所需的最小权限，数据库连接不使用 root/sa。
3. **安全配置默认化**：框架和中间件的安全选项默认为开启，需要显式决定关闭。
4. **持续更新依赖**：使用 `npm audit` / `pip audit` / `dependabot` 定期扫描，SLA 内修复已知漏洞。

## 检查清单
- [ ] 所有用户输入是否在服务端做了二次校验（不依赖前端验证）？
- [ ] 数据库查询是否全部使用参数化（无字符串拼接 SQL）？
- [ ] 是否有越权漏洞（水平越权/垂直越权）的测试覆盖？
- [ ] 第三方依赖是否通过安全扫描无高危漏洞？
- [ ] 敏感操作是否有审计日志记录（谁、何时、做了什么）？
