# 这是什么

上下文预算管理是在使用大语言模型时，系统性地分配和管理有限的上下文窗口资源的方法论。合理的预算管理直接影响输出质量、响应速度和成本。

# 何时使用

- 搭建基于 LLM 的应用或 Agent 系统
- 处理大型代码库或多文件项目时
- 长时间对话积累了过多历史
- API 成本超出预算需要优化
- 设计 Agent 系统的上下文策略

# 核心规则

1. **上下文是稀缺资源**：越接近上下文窗口上限，模型表现越差。预留至少 20% 空间用于推理。
2. **相关性优先于完整性**：放进去的每一条信息都应直接服务于当前任务。不是"可能有用的都放"，而是"没它不行的才放"。
3. **三段分配法**：将上下文窗口划分为 System（系统指令）、History（对话历史）、Task（当前任务数据）三部分，各自有预算上限。
4. **压缩保留语义**：压缩不是截断。在减少 token 数量的同时，必须保留关键信息、决策逻辑和约束条件。
5. **模型与任务匹配**：轻量任务用低成本模型，复杂推理用高能力模型。不要用大炮打蚊子。

# 三段分配模型

```
|←────────────── 上下文窗口 ──────────────→|
|  System  |    History    |    Task      |
|  固定区  |   滚动窗口    |   弹性区      |
```

- **System 区**（建议不超过 20%）：系统提示词、角色设定、全局规则、安全约束。内容相对稳定，不随对话变化。
- **History 区**（建议 30%-40%）：最近的对话记录，包括用户消息、助手回复、工具调用结果。采用滑动窗口，老旧信息被压缩或丢弃。
- **Task 区**（建议 40%-50%）：当前任务需要的文件内容、代码片段、数据表格、参考文档。任务完成后清空。

# 压缩触发条件

当上下文使用率达到以下阈值时触发压缩：

- **75%（警戒线）**：开始压缩历史对话中的冗余内容（合并重复信息、移除未使用的工具结果）
- **85%（行动线）**：将早期详细对话压缩为结构化摘要，保留决策、代码变更、错误及修复
- **90%（紧急线）**：仅保留最近几轮对话的原始内容，其余全部替换为摘要

压缩原则：
- 保留决策依据：为什么选择 A 方案而不是 B 方案
- 保留代码变更：修改了什么文件，关键代码片段
- 保留待解决问题：未完成的任务、未解决的 Bug
- 保留约束条件：用户偏好、项目规范、环境限制

# 模型选择成本

| 任务类型 | 推荐模型级别 | 原因 |
|----------|-------------|------|
| 简单问答、格式转换 | 轻量/Haiku 级 | 任务简单，高质量模型过度服务 |
| 代码审查、Bug 修复 | 标准/Sonnet 级 | 需要一定推理，但不过度复杂 |
| 架构设计、多文件重构 | 高级/Opus 级 | 需要深度推理和全局视角 |
| Agent 调度、任务分解 | 标准级 | 拆分任务本身不需要最强模型 |

选择依据：
- 如果轻量模型能完成 90% 的质量，就不要用高级模型
- 如果任务失败需要重试，先用轻量模型尝试，再升级
- 系统指令应指导模型在不确定时请求升级而非猜测

# Profile 预算建议

不同使用场景的推荐预算上限：

- **交互式对话**：单次交换不超过 4K tokens 输入，确保响应速度
- **代码补全**：上下文 2K-8K tokens，精准聚焦当前文件和导入
- **项目级重构**：上下文 32K-64K tokens，包含架构文件和关键代码
- **全库分析**：上下文 100K+ tokens，需要摘要和索引辅助
- **长时间会话**：主动监控 token 使用，在 75% 时触发压缩

# 优化技巧

- **文件摘要替代完整内容**：当文件超过 500 行时，先用摘要描述结构和关键逻辑
- **索引式加载**：先加载文件列表和符号表，再按需加载具体函数体
- **增量更新**：只发送差异部分而非完整文件
- **结构化输出**：要求模型输出 JSON 格式时，明确 schema 减少冗余描述
- **避免重复**：相同信息不重复发送，工具调用结果中已包含的内容不再在 prompt 中赘述

# 内容优先级排序

不确定哪些内容该放进去时，按以下优先级筛选：

1. **任务定义**（最高优先级）：要做什么，不要做什么，输出格式
2. **约束条件**：不能违反的规则，如安全策略、代码风格
3. **当前相关代码**：与任务直接相关的文件内容
4. **上下文依赖**：理解当前代码所需的接口定义、类型声明
5. **历史决策**：之前做过的相关决策及原因
6. **参考示例**：类似的实现或期望的输出样式
7. **辅助信息**（最低优先级）：文档链接、背景知识

# 压缩策略详解

- **摘要替代**：将大段文字压缩为结构化摘要，关键信息保留
- **去重合并**：相同或高度相似的信息只保留一份
- **丢弃无用**：工具调用结果中未被后续引用的部分可以安全丢弃
- **渐进加载**：先给概要，模型请求时再给细节
- **差分表示**：文件变更只发送差异，而非完整文件

# 成本估算方法

- **输入 token 成本** = 输入 token 数 × 模型输入单价
- **输出 token 成本** = 输出 token 数 × 模型输出单价（含推理 token）
- **缓存命中节省**：某些模型对重复的 prompt 前缀提供缓存折扣
- **总成本** = 输入成本 + 输出成本 - 缓存节省
- **估算公式**：英文约 1 token ≈ 0.75 词，中文约 1 token ≈ 1.5 字
