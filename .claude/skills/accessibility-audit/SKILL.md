# 这是什么

无障碍审计是评估数字产品对所有用户（包括残障人士）可用性的过程。良好的无障碍设计不仅满足合规要求，也提升了所有用户的体验。

# 何时使用

- 新功能上线前的可访问性检查
- 定期审计（建议每个大版本一次）
- 收到无障碍相关的用户投诉
- 法律法规合规审查
- 设计系统组件的标准化检查

# 核心规则

1. **无障碍不是事后补丁**：在设计阶段就考虑无障碍，比在开发完成后修补成本低十倍以上。
2. **语义化 HTML 优先于 ARIA**：原生 HTML 元素的语义已经提供了大量无障碍支持。只有 HTML 不足时才使用 ARIA 补充。
3. **所有交互必须键盘可达**：不能用鼠标的人，应该能用键盘完成所有操作。Tab 导航，Enter/Space 激活，Escape 关闭。
4. **信息不以单一感官通道传递**：颜色、声音、位置等信息必须有对应的文本替代。图片有 alt，视频有字幕，图标有标签。
5. **测试不能只靠自动化工具**：自动工具只能检测约 30% 的无障碍问题。人工测试不可或缺。

# 工作流程

1. **自动化扫描**
   - 使用 axe-core 或 Lighthouse 进行快速扫描
   - 检测缺失的 alt 属性、对比度不足、ARIA 误用等自动可检问题
   - 修复所有自动化工具报告的明确问题

2. **键盘导航测试**
   - 断开鼠标，仅用键盘完成所有页面核心操作
   - 检查焦点顺序是否合理，不应跳来跳去
   - 检查焦点是否始终可见（有可见的焦点指示器）
   - 检查是否有键盘陷阱（进入后无法 Tab 离开）
   - 确认模态框打开时焦点移入，关闭时焦点回到触发元素

3. **屏幕阅读器测试**
   - 使用至少一种主流屏幕阅读器测试（NVDA/Windows、VoiceOver/macOS）
   - 检查页面标题、地标区域是否正确识别
   - 检查表单控件的标签是否与控件正确关联
   - 检查动态内容更新是否被正确播报
   - 检查图片的替代文本是否传达了同等信息

4. **视觉检查**
   - 文本对比度检查：正文 4.5:1 以上，大文本 3:1 以上
   - 色盲友好：信息不仅靠颜色区分，配合图标或文字
   - 缩放测试：200% 缩放后内容不重叠、不丢失
   - 间距检查：可点击区域不小于 44x44px（移动端）

5. **输出审计报告**
   - 按 WCAG 成功标准逐条评估
   - 标注合规级别：A（最低）、AA（标准）、AAA（最高）
   - 每项注明检测结果：通过、失败、不适用、需人工判断
   - 失败项附带修复建议和代码示例

# WCAG 2.2 核心检查清单

## A 级（必需）

- 非文本内容有替代文本（1.1.1）
- 音频和视频有字幕或文字稿（1.2.1-1.2.3）
- 信息不以颜色为唯一区分方式（1.4.1）
- 所有功能可键盘操作，无键盘陷阱（2.1.1-2.1.2）
- 页面有描述性标题（2.4.2）
- 链接目标可从链接文本理解（2.4.4）
- 当前语言有标记（3.1.1）
- 输入错误有文字说明（3.3.1）
- 标签和说明在用户输入时可见（3.3.2）

## AA 级（推荐）

- 文本对比度至少 4.5:1（1.4.3）
- 文本可缩放到 200% 不丢失内容（1.4.4）
- 图像不用作文本内容（1.4.5）
- 多种方式定位页面（2.4.5）
- 可见焦点指示器（2.4.7）
- 一致导航（3.2.3）
- 输入错误时有修改建议（3.3.3）
- 法律/财务数据提交可撤销或确认（3.3.4）

# ARIA 规范

- **规则一**：能不用的地方不用 ARIA。原生 HTML 元素已经内置了正确的 ARIA 语义。
- **规则二**：不要用 ARIA 改变原生语义。`<button role="heading">` 是错的。
- **规则三**：交互式 ARIA 控件必须可用键盘操作。
- **规则四**：不要给可见且可聚焦的元素添加 `role="presentation"` 或 `aria-hidden="true"`。
- **规则五**：所有交互元素必须有可访问的名称（通过标签、aria-label 或 aria-labelledby）。

# 焦点管理最佳实践

- 页面加载时将焦点放在主要内容区域的第一个可交互元素或 skip link 目标
- 模态框/对话框打开时，焦点移到对话框内第一个可聚焦元素
- 模态框/对话框关闭时，焦点回到触发它的元素
- 动态插入内容时，将焦点移到新内容的起始位置
- 单页应用路由切换后，将焦点移到新页面的标题或主内容
- `tabindex="0"` 让元素进入 Tab 序列，"-1" 让元素可脚本聚焦但不进入 Tab 序列

# 对比度标准速查

| 元素 | 最低对比度 | WCAG 级别 |
|------|-----------|-----------|
| 正文文本 | 4.5:1 | AA |
| 大文本（18px+ 粗体或 24px+） | 3:1 | AA |
| 图标和图形 | 3:1 | AA |
| 纯粹装饰元素 | 无要求 | — |
| 正文文本（增强） | 7:1 | AAA |
| 大文本（增强） | 4.5:1 | AAA |
