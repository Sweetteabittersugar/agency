# 网络搜索 + 交叉验证

## 用途
在需要外部信息时，从多个来源获取并交叉验证，返回带引用来源的综合结果。

## 数据源优先级

| 优先级 | 来源 | 适用场景 | 信任度 |
|--------|------|---------|:---:|
| 1 | 官方文档 (Context7) | API/框架用法 | ⭐⭐⭐⭐⭐ |
| 2 | GitHub Issues/Discussions | Bug/兼容性问题 | ⭐⭐⭐⭐ |
| 3 | Stack Overflow | 常见编程问题 | ⭐⭐⭐ |
| 4 | 开发者博客/Medium | 最佳实践/教程 | ⭐⭐⭐ |
| 5 | 通用搜索引擎 | 广泛信息查询 | ⭐⭐ |

## 搜索策略

### Step 1: 查询分析
- 解析用户意图：API 用法 / Bug 排查 / 最佳实践 / 新闻 / 通用信息
- 提取关键术语和技术栈
- 确定目标数据源优先级

### Step 2: 并行搜索
- 至少从 2 个不同来源获取信息
- 每个来源提取关键断言/代码/数据
- 记录来源 URL 和获取时间

### Step 3: 交叉验证
- 将多个来源的断言逐条对比
- 标记一致/冲突/唯一的信息
- 评估各来源的权威性和时效性

## 验证矩阵

| 状态 | 含义 | 输出标注 |
|:---:|------|---------|
| ✅ 一致 | 2+ 个独立来源确认 | 直接使用 |
| ⚠ 部分一致 | 仅 1 个来源，或来源部分冲突 | 标注"单一来源" |
| ❌ 冲突 | 2+ 个来源给出矛盾信息 | 列出双方，建议人工判断 |
| 🔄 过时 | 信息超过 2 年且可能已变更 | 标注"可能过时(last update: YYYY-MM)" |

## 搜索引擎选择

### Context7 (优先 — 实时官方文档)
触发条件: 涉及特定库/框架时
```python
# 先在 Context7 解析 library ID, 再查询
context7.resolve(library_name) → context7.query(library_id, query)
```

### 通用搜索 (WebSearch)
触发条件: 无特定库/框架，或需要广泛信息
- 包含当前年份确保时效性
- 使用 `site:` 过滤可信域名
- 避免 `.md` / 内容农场

## 核心规则
- 每个查询至少交叉验证 2 个来源
- 冲突信息 → 标注所有来源和观点，不自行裁决
- 过时信息 → 标注最后更新时间，建议验证
- 输出末尾必须附 `Sources:` 列表（URL + 简短描述）
- 代码示例优先使用官方文档来源
- 与 Profile 联动：minimal 单源 3 结果，standard 双源各 5 结果，full 三源各 10 结果
