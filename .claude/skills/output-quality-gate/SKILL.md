# 输出质量门

## 用途
在输出最终结果前执行自动质量检查，确保格式合规、内容完整、无安全问题。

## 核心规则
- 格式检查：确认输出符合约定的格式模板（如 STATUS 行 + ## 用户摘要）
- 完整性验证：关键字段非空（`data`、`summary`），文件路径为绝对路径
- 安全检查：输出不包含密钥、token、会话 ID、内部路径等敏感信息
- 代码块语言标记完整，文件路径可点击跳转
- 质量门不通过→退回源头重新生成，最多重试 3 次
