# 重构模式

## 用途
提供安全重构的模式指南，确保每次重构可逆、可验证、不影响现有功能。

## 核心规则
- 每次重构只做一件事：提取方法、重命名、消除重复，互不混在一起
- 重构前后行为保持一致，先确认有测试覆盖再动手
- 小步提交：每个独立重构步骤一个 commit，方便回滚和 review
- 提取的公共逻辑放入独立的工具函数/模块，避免循环依赖
- 重构完成后运行完整测试套件，确认零回归
