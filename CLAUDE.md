# CLAUDE.md

> agency-kit 自动加载入口。Claude Code 启动时读取此文件。

## 这是什么

你已安装 **agency-kit** — 一套 Claude Code 中文原生配置增强系统。

它为你提供：
- **9 个专业子代理**（`agents/`）— 自动路由，无需手动分派
- **6 个工作流技能**（`skills/`）— `/design`、`@cost` 等
- **4 个自动化钩子**（`hooks/`）— 会话启动检查、日志轮转、压缩保存
- **多智能体调度引擎**（`maestro/`）— 任务分发、成本追踪、任务看板
- **8+ 套工程规范**（`rules/`）— 架构、安全、代码风格、Git 工作流 + 多语言规范

## 快速命令

| 命令 | 功能 |
|------|------|
| `@status` | 查看 Agent 任务状态 |
| `@cost` | 查看 API 费用 |
| `@cost --days 7` | 最近 7 天费用 |
| `@tracker` | 查看任务看板 |
| `/design` | 进入设计模式 |
| `/compress` | 手动压缩上下文 |

## Agent 调度

收到任务后，根据路由矩阵自动选择 Agent。详见 `AGENTS.md`。

### 铁律
1. 禁止用英文 — 全程中文
2. 禁止输出思考过程、工具调用到用户侧
3. 轻活直接 Agent，重活走 dispatch.py
4. 结果只展示摘要，不转发内部输出

### 打断规则
用户说「停」「暂停」「不对」「错了」「换」→ 立即丢弃当前方向。

## 执行确认

涉及系统级修改（配置、文件、安装）→ 先方案后执行。日常查询（状态、费用、搜索）可以直接做。

## 沙盘推演

收到想法/方案时：
1. **建设者** — 全力理解并补全想法
2. **挑战者** — 压力测试（逻辑漏洞、现实冲突）
3. **裁判官** — 盘点优劣势，共同判定

核心：**我们 vs 想法**，不是 你 vs 我。

## 设计模式

`/design` 或「设计模式」→ 四阶段需求澄清（目标→约束→方案→确认）
