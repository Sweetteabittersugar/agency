# 安全策略

> 最后更新：2026-06-05

## 报告安全漏洞

如果你发现了安全漏洞，**请不要在公开 Issue 中报告**。请通过以下方式私密报告：

- **邮件**：向项目维护者发送加密邮件（GitHub 主页查看联系方式）
- **GitHub 私密报告**：在仓库 Security 标签页 → "Report a vulnerability"

请在报告中包含以下信息，帮助我们快速复现和修复：

1. 漏洞的详细描述
2. 复现步骤
3. 受影响的版本
4. 可能的修复建议（如有）

## 我们的承诺

| 阶段 | 时间承诺 |
|------|----------|
| 确认收到报告 | **24 小时内** |
| 初步评估与确认 | **3 个工作日内** |
| 发布修复 | **7 个工作日内**（严重漏洞） |
| 公开披露 | 修复发布后，与报告者协商确定 |

我们承诺：
- 在 24 小时内确认收到你的报告
- 保持透明：在修复过程中与你保持沟通
- 给予致谢：在修复公告中署名（如你同意）
- 不采取法律行动：对于善意、负责任的披露

## 安全最佳实践

作为使用者，请遵循以下建议：

1. **API Key 保护**：never 将 API Key 硬编码在源码或配置文件中。使用环境变量或密钥管理器。
2. **依赖更新**：定期运行 `pip list --outdated` 检查 Python 依赖，及时更新。
3. **权限最小化**：Agent 执行时使用最小必要权限，尤其是涉及文件删改和系统配置的操作。
4. **脚本审查**：安装前审查 `install.sh` / `install.ps1` 中的操作——它们会修改你的 `~/.claude/` 配置。
5. **沙箱隔离**：敏感或不可信任务使用 `maestro/sandbox.py` 隔离执行。

## 范围

本安全策略涵盖：
- `agency-kit` 仓库中的所有代码和配置
- Maestro 调度引擎（`maestro/` 目录下的 Python 脚本）
- 安装脚本（`install.sh`、`install.ps1`）
- Agent 和 Skill 定义文件

本安全策略不涵盖：
- 第三方依赖的已知漏洞（请向对应上游项目报告）
- Claude Code 本身的安全问题（请向 Anthropic 报告）
