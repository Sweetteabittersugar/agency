# Agency 需求规格

> 版本: 0.1.0 | 最后更新: 2026-06-10

## 项目目标

Agency = Claude Code 的 Web UI 面板。让用户通过浏览器操控 Claude Code，32 个 AI Agent 协同工作。

## 核心功能（按优先级）

### P0 — 必须有（产品可用）

| ID | 功能 | 描述 | 验收标准 |
|----|------|------|----------|
| REQ-001 | Web 聊天界面 | 浏览器中与 Agent 对话 | 输入文本 → 发送 → 流式返回 Agent 回复 |
| REQ-002 | Agent 路由 | 自动将任务分给正确的 Agent | 命中率 > 80%，低置信走通用 Agent |
| REQ-003 | 多模型切换 | 支持 DeepSeek/Anthropic/OpenAI 等 | Provider 配置后可切换 |
| REQ-004 | API Key 管理 | 安全存储和传递 API Key | Key 不入日志、不入 git、不入前端 |
| REQ-005 | 安装部署 | 一键安装，开箱即用 | install.bat/sh 执行后 agency start 可用 |

### P1 — 应该有（体验完整）

| ID | 功能 | 描述 | 验收标准 |
|----|------|------|----------|
| REQ-101 | 多面板分屏 | 同时查看多个 Agent 工作 | 支持 1/2/4 面板布局 |
| REQ-102 | 费用追踪 | 实时显示 API 调用费用 | 仪表盘显示日/周/月费用 |
| REQ-103 | Agent 工厂 | 用户可创建自定义 Agent | 向导式创建 + 自动生成 .md |
| REQ-104 | Skill 管理 | 管理 Agent 的技能模板 | 启用/禁用/编辑 Skill |
| REQ-105 | 会话持久化 | 刷新页面不丢对话 | JSONL 存储 + 快照压缩 |
| REQ-106 | Worktree 隔离 | Agent 在隔离目录运行 | 创建/删除 worktree |
| REQ-107 | 命令面板 | 键盘快捷操作 | Ctrl+K 打开，模糊搜索 |

### P2 — 可以有（锦上添花）

| ID | 功能 | 描述 |
|----|------|------|
| REQ-201 | Demo 模式 | 免 Key 体验基本功能 |
| REQ-202 | 移动端适配 | 响应式布局 |
| REQ-203 | 配置导入导出 | 备份和迁移配置 |
| REQ-204 | MCP 面板 | 可视化管理 MCP 服务器 |

## 用户角色

| 角色 | 描述 |
|------|------|
| 开发者 | 主要用户，用来写代码、调试、审查 |
| 技术管理者 | 查看费用、分配任务、监控进度 |
| 内容创作者 | 写作、翻译、文档生成 |

## 非功能需求

| ID | 类型 | 要求 |
|----|------|------|
| NFR-001 | 性能 | 路由决策 < 500ms，SSE 流延迟 < 100ms |
| NFR-002 | 安全 | API Key 不入日志/git/前端内存 |
| NFR-003 | 可用性 | 网络错误不崩溃，优雅降级 |
| NFR-004 | 可维护性 | 模块化架构，单文件 < 800 行 |
| NFR-005 | 兼容性 | Python 3.10+，Windows/macOS/Linux |

## 版本路线

- v0.1.0: 基础 Web 面板 + Agent 路由
- v0.2.0: 多面板分屏 + Worktree + 会话持久化
- v0.3.0: 命中率优化 + 仪表盘完善
- v1.0.0: 生产就绪 — 安全审计 + 性能基准 + 完整测试
