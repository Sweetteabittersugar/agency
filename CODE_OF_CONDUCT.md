# 贡献者行为准则

> 版本 2.1 — 采用 [Contributor Covenant](https://www.contributor-covenant.org/zh-cn/version/2/1/code_of_conduct/)

## 我们的承诺

为了营造一个开放、包容且富有建设性的社区，我们作为贡献者与维护者郑重承诺：无论年龄、体型、残障状况、民族、性别认同与表达、经验水平、教育程度、社会经济地位、国籍、个人外貌、种族、宗教或性取向如何，参与本项目及相关社区的过程对所有人免于骚扰。

我们承诺以建设社区的方式行事与互动，始终以项目长远健康发展为优先。

具体来说，我们承诺：

- 对贡献者保持尊重和耐心，无论其经验水平如何
- 为新成员提供友好、欢迎的入门体验
- 以建设性态度回应批评——把每一条反馈都当作让项目更好的机会
- 当分歧出现时，聚焦于技术问题本身，不针对个人

## 行为标准

### 鼓励的行为

有助于营造积极社区环境的行为包括但不限于：

- 友善待人，对他人抱有同理心
- 尊重不同观点、经历和立场
- 给予并优雅地接受有建设性的反馈
- 为自己的错误承担责任，并从中学习
- 不仅关注个人利益，也着眼于整个社区
- 使用包容性语言，避免排他性表述

### 不允许的行为

不可接受的行为包括但不限于：

- 使用带有性暗示的语言或图像，以及任何形式的性关注或性骚扰
- 挑衅、侮辱或贬损性评论，以及人身或政治攻击
- 公开或私下骚扰
- 未经明确允许，公开他人的私人信息（如物理地址或电子邮箱）
- 在专业场合中被合理认定为不当的其他行为

## 执行准则

社区维护者有责任阐明并执行可接受行为的标准，并应针对任何不可接受的行为采取适当且公平的纠正措施。

维护者有权且负责删除、编辑或拒绝不符合本行为准则的评论、提交、代码、议题（Issue）及其他贡献，并在适当时告知做出该行为的个人或团队决定的原因。

## 适用范围

本行为准则适用于一切社区空间，亦适用于个人在公共空间正式代表社区之时。代表本社区的情形包括使用官方邮箱地址、通过官方社交媒体账号发布内容、或在线上线下活动中担任指定代表。

## 报告途径

可接受行为和不可接受行为的实例均可通过以下方式报告给项目维护者：

- **GitHub Discussions**（公开讨论）：适合一般性问题或建议
- **邮件**（私密报告）：[项目维护者邮箱或 GitHub 主页联系方式]
- 在 GitHub 仓库页面使用"Report"按钮进行举报

所有投诉将被及时、公正地审查和调查。报告者的隐私和安全将被优先保护。

处理流程：

1. **受理**：维护者在 48 小时内确认收到报告
2. **调查**：收集相关证据，与当事各方沟通（如需要）
3. **裁定**：根据本准则的执行分级标准确定处理方案
4. **执行**：实施相应措施，并以适当方式告知结果
5. **反馈**：如报告者对处理结果不满意，可请求复审

## 执行分级

维护者将遵循以下社区影响后果指南，确定违反本准则的后果：

| 级别 | 违规程度 | 后果 |
|------|----------|------|
| **纠正** | 不当语言或其他被认为不专业的行为 | 一次私下书面警告，说明违规性质及原因。可能要求公开道歉。 |
| **警告** | 单次事件或一系列行为构成违规 | 警告并附上一段时间的冻结期。其间禁止与相关人员互动，包括主动和被动。违反可能导致临时或永久禁入。 |
| **临时禁入** | 严重违反社区标准，包括持续不当行为 | 指定时间内禁止与社区进行任何形式的互动或公开交流。其间不得与相关人员进行公开或私下互动。违反可能导致永久禁入。 |
| **永久禁入** | 表现出违反社区标准的系统性问题，包括持续不当行为、骚扰个人或攻击/贬低群体 | 永久禁止与社区进行任何形式的公开互动。 |
